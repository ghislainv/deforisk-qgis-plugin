Directrices comunitarias#

El complemento deforisk para QGIS es de código abierto y está publicado bajo la licencia GNU GPL versión 3. Cualquiera que esté interesado puede contribuir al desarrollo del paquete. Hay muchas maneras de contribuir, como escribir tutoriales, ejemplos o pruebas, mejorar la documentación, enviar informes de errores y peticiones de características, o escribir código para proporcionar nuevas funcionalidades que puedan ser incorporadas en futuras versiones del paquete. Todos los colaboradores deben comprometerse a seguir el Código de conducta del proyecto <code_of_conduct.html>`_.

Informar de un problema#

Si desea informar de un error, solicitar una función o debatir un problema, abra una issue en la página del proyecto en GitHub.

Contribuir al código#

Los cambios en el código fuente y la documentación deben realizarse a través de pull request de GitHub (PR).

Puedes aprender a hacerlo en esta serie de vídeos gratuitos Cómo contribuir a un proyecto de código abierto en GitHub, el tutorial sobre el flujo de trabajo git de Aaron Meurer, o la guía Cómo contribuir al código abierto.

Los pasos importantes a seguir son:

  1. Comience por crear un fork del repositorio deforisk-qgis-plugin.

  2. Realice cambios en el código fuente en una rama de desarrollo, no en la rama main por defecto.

  3. Mantén las ramas principal y de desarrollo de tu fork actualizadas con los cambios en el repositorio deforisk-qgis-plugin.

  4. Confirma los cambios realizados. Chris Beams ha escrito una guía sobre cómo escribir buenos commit.

  5. Empuja a tu fork y envía un pull request.
