Charte communautaire#

Le plugin QGIS deforisk est Open Source et publié sous la licence GNU GPL version 3. Toute personne intéressée peut contribuer au développement du paquet. Il y a de nombreuses façons de contribuer, comme l’écriture de tutoriels, d’exemples ou de tests, l’amélioration de la documentation, la soumission de rapports de bugs et de demandes de fonctionnalités, ou l’écriture de code pour fournir de nouvelles fonctionnalités qui peuvent être incorporées dans les futures versions du paquet. Chaque contributeur doit accepter de suivre le Code de conduite du projet.

Signaler un problème#

Si vous souhaitez signaler un bug, demander une fonctionnalité ou discuter d’un problème, veuillez ouvrir une issue sur la page du projet GitHub.

Contribuer au code#

Les modifications du code source et de la documentation doivent être effectuées par le biais de pull requests (PR) sur GitHub.

Vous pouvez apprendre à le faire grâce à cette série de vidéos gratuites Comment contribuer à un projet Open Source sur GitHub, le tutoriel sur le flux de travail git d’Aaron Meurer, ou le guide Comment contribuer à l’Open Source.

Les étapes importantes à suivre sont les suivantes :

  1. Commencez par créer un fork du dépôt deforisk-qgis-plugin.

  2. Apporter des modifications au code source sur une branche de développement, et non sur la branche main par défaut.

  3. Gardez les branches principale et de développement de votre fork à jour avec les changements dans le dépôt deforisk-qgis-plugin.

  4. Validez les changements que vous avez effectués. Chris Beams a écrit un guide sur la façon d’écrire de bons messages de commit.

  5. Poussez vers votre fork et soumettez une pull request.
